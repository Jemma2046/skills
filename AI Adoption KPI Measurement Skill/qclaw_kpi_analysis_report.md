**根因分析 (5-Why)**:
1. **为什么提示词质量不稳定？** → 未使用结构化提示词模板
2. **为什么未使用模板？** → 不知道有哪些模板可用

4. **为什么未系统学习？** → 专注于功能实现，忽视方法论
5. **根本原困**: **重功能、轻方法**，缺乏提示词工程系统训练

**改进方案**:
- **P0**: 学习并使用结构化提示词模板 (提供模板库)
- **P1**: 每次交互前，先写提示词大纲

## 📝 可直接使用的提示词模板

### 模板 1: 任务执行类

```markdown
# 角色
你是一个 [角色名称，如：Python 开发专家]

# 任务
[明确描述任务，如：编写一个 Python 脚本，从 URL 抓取 JD]

# 上下文
- 当前项目: [项目名]
- 现有技术栈: [如：Python, BeautifulSoup, requests]
- 约束: [如：需要绕过反爬虫]

# 要求
1. [要求 1，如：使用 HEADLESS 浏览器]
2. [要求 2，如：添加随机延迟]
3. [要求 3，如：错误处理]

# 输出格式
- 代码文件: [文件名]
- 说明文档: [README.md]
- 示例输出: [示例]

# 示例 (可选)
[提供一个理想输出的示例]
```

### 模板 2: 分析类

```markdown
# 角色
你是一个 [角色，如：数据分析师]

# 任务
分析 [对象，如：qclaw 工具使用情况]

# 分析框架
使用 [框架，如：AI Adoption KPI Measurement Skill 的 4 维 KPI]

# 数据来源
- [来源 1，如：MEMORY.md]
- [来源 2，如：HEARTBEAT.md]

# 输出要求
1. KPI 评分 (0-100)
2. 差距分析
3. 改进建议 (优先级排序)
4. 个性化学习计划

# 输出格式
Markdown 报告，包含表格和可视化建议
```
